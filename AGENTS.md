# AGENTS.md

## Төслийн зорилго
- Энэ repo дээр хотын талбайн ажиллагаанд зориулсан Odoo болон web workspace хөгжүүлдэг.
- `municipal_project_mobile` модуль нь талбайн төслийн удирдлагыг mobile-first логикоор хэрэгжүүлнэ.

## Ажлын байр суурь
- UI өөрчлөлт хийхдээ зөвхөн coder шиг биш, product designer, UX strategist, QA engineer, бодит end-user-ийн өнцгөөс зэрэг ажилла.
- Замбараагүй, ойлгомжгүй, чухал мэдээллийг нуусан, хэт их бодолт шаардсан UI-г хүлээн зөвшөөрөхгүй.
- Менежер 5 секундийн дотор `өнөөдөр юу болсон`, `юу хоцорсон`, `юунд арга хэмжээ хэрэгтэй` гэдгийг ойлгох ёстой.

## Гол дүрэм
- End-user UI текстийг зөвхөн Монгол хэлээр бичнэ.
- ERP-style олон таб, хэт их талбар, давхар урсгалаас зайлсхийнэ.
- Нэг дэлгэц дээр нэг гол зорилго баримтална.
- Үйл ажиллагаа хариуцсан менежерт executive summary, төслийн менежерт хяналтын түвшний мэдээлэл, багийн ахлагчид гүйцэтгэлийн урсгал, ажилтанд зөвхөн унших горим өгнө.
- Mobile UX дээр kanban, stat button, sticky action bar-ийг түрүүлж ашиглана.
- Horizontal scroll гаргахгүй.
- Текст уншихад амархан, зай хэмжээ тогтвортой, товчлуурууд эрхий хуруунд хүрэхэд хялбар байна.
- Team leader нийтлэг ажлаа 3 товшилтоос дотроо хийж чаддаг байна.

## Dashboard тэргүүлэх бүтэц
- Top KPI заавал харуул: өнөөдрийн цуглуулсан кг, идэвхтэй машин, идэвхгүй машин, хоцорсон ажил, гүйцэтгэлийн хувь.
- Ажлын ангиллыг салгаж харуул: айл өрх, аж ахуйн нэгж, нийтийн талбай.
- Тээврийн хэрэгслийн гүйцэтгэлийг салгаж харуул: өндөр гүйцэтгэлтэй, сул гүйцэтгэлтэй, идэвхгүй.
- Alert блок заавал байлга: хоцорсон маршрут, тайлан дутуу, идэвхгүй ажилтан.
- Хамгийн чухал action болон анхаарах асуудлыг first screen дээр тод харуул.

## Role hierarchy
1. Үйл ажиллагаа хариуцсан менежер
2. Төслийн менежер
3. Багийн ахлагч
4. Ажилтан

## Хөгжүүлэлтийн зарчим
- Odoo модуль install-ready байх.
- Security group, access, record rule гурав хоорондоо зөрчилдөхгүй байх.
- Worker write эрхгүй үлдэнэ.
- Тайлангийн урсгал `ноорог -> илгээсэн -> баталсан/буцаасан` логиктой байна.
- Desktop болон mobile аль алинд нь hierarchy, spacing, alignment, overflow, consistency-г алдагдуулахгүй.
- Менежер, ахлагч, ажилтан бүрийн screen нь тухайн role-ийн шийдвэр гаргалт эсвэл гүйцэтгэлд шууд үйлчлэх ёстой.

## UI өөрчлөлт бүрийн дараах заавал хийх урсгал
1. `lint`, `typecheck`, `tests` ажиллуул.
2. Аппаа асаа.
3. Browser нээ.
4. Dashboard, list page, detail page-уудаар ор.
5. Desktop болон mobile screenshot ав.
6. Дараах шалгалтыг хий:
   - Visual: spacing, alignment, overflow, consistency
   - UX: clarity, confusion, missing actions, poor grouping
7. Илэрсэн асуудлыг зас.
8. Дахин test хийж, browser дээр нягтал.

## Browser болон тестийн дүрэм
- Playwright байхгүй бол тохируулж нэм.
- Дор хаяж dashboard нээх, navigation дарах, үндсэн flow шалгах, screenshot авах тесттэй бай.
- UI зөв харагдаж байвал л ажил дууссанд тооц; зөвхөн build амжилттай болсноор дууссан гэж үзэхгүй.
